# Contexto do Produto

Este documento preserva e organiza as respostas discutidas com o professor, ajustadas ao estado atual do ParaBula.

## Para quem é o produto?

O foco principal recomendado é B2B: profissionais da saúde, clínicas, hospitais, farmácias digitais, telemedicinas e sistemas que precisam validar prescrições ou dispensações.

Prós do B2B:

- mercado mais estável;
- maior disposição a pagar;
- possibilidade de integração com EMR/EHR e sistemas de prescrição;
- maior aderência ao uso de API.

Contras do B2B:

- ciclo de venda mais longo;
- necessidade de validação clínica;
- exigência de segurança, rastreabilidade e conformidade.

Um uso B2C pode existir futuramente, mas exige mais cuidado com linguagem, responsabilidade legal e orientação clara de que o sistema não substitui avaliação profissional.

## Qual dor específica ele resolve?

O ParaBula ataca o risco de eventos adversos evitáveis causados por prescrições ou administrações de medicamentos sem checagem adequada do contexto clínico do paciente.

Essa dor aparece em quatro camadas:

- interações medicamentosas perigosas entre dois ou mais fármacos;
- contraindicações ligadas ao perfil do paciente, como idade, sexo biológico, gravidez, lactação, comorbidades e condições metabólicas;
- dificuldade de acessar rapidamente informação confiável da bula no ponto de decisão;
- risco por via de administração incorreta, funcionalidade planejada para as próximas etapas do projeto.

## O que torna a proposta diferente?

O diferencial é combinar bulas oficiais como fonte de dados, API reutilizável e IA generativa para produzir uma resposta contextualizada.

Em vez de apenas retornar um alerta genérico, o ParaBula tenta explicar:

- quais medicamentos estão envolvidos;
- qual tipo de risco foi identificado;
- qual informação da bula sustenta a análise;
- como o perfil do paciente muda a interpretação do risco.

## Como a startup ganharia dinheiro?

Modelo proposto:

- API por uso para empresas, farmácias digitais, telemedicinas e softwares de saúde;
- planos de assinatura B2B para clínicas, hospitais e integrações com suporte;
- plano enterprise com SLA, auditoria, integração com sistemas internos e requisitos de conformidade;
- eventualmente, uma versão B2C limitada, com consultas gratuitas e recursos premium.
